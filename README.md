## Repository Structure

The repository is organized into the following folders and files to ensure modularity and maintainability:

### `src/`
The main source folder for the application. It contains all the code, components, pages, services, and styles used in the project.

#### `components/`
- Contains reusable React components used across the application.
- Examples: `quiz.js`, `quizBoxes.js`, `MainContent.js`.
- Handles UI elements like quiz boxes, navigation, and quiz views.

#### `pages/`
- Contains React components that represent different pages of the application.
- Examples: `welcome.js`, `StudyRoom.js`, `store.js`, `spacedLearningQuiz.js`.
- Each file corresponds to a specific route or feature in the app, such as the study room, store, or spaced learning quiz.

#### `services/`
- Contains utility functions and services for interacting with external systems like Firebase.
- Examples: `addQuiz.js`, `userSearch.js`, `showPoints.js`.
- Handles database operations, user management, and other backend-related tasks.

#### `styles/`
- Contains CSS files for styling the application.
- Example: `index.css`.
- Defines the look and feel of the app, including layout, colors, and typography.

#### `models/`
- Contains data models or schemas for the application.
- Examples: `quiz.js`, `questions.js`, `db.js`.
- Defines the structure of data like quizzes, questions, and user-related information.

#### `utils/`
- Contains utility functions or helpers used throughout the app.
- Examples: Firebase configuration, reusable functions, etc.
- Example: `unfinishedfirebaseDB.js` contains Firebase initialization and database-related logic.

### Additional Directories

#### `public/`
- Contains static assets and files that are publicly accessible.
- Examples: `index.html`, `favicon.ico`.
- Used for meta tags, app icons, and other static resources.

#### `node_modules/`
- Contains all the dependencies installed via npm or yarn.
- Automatically generated by the package manager based on `package.json`.

#### `.github/`
- Contains GitHub-specific configuration files, such as workflows for CI/CD.
- Example: `deploy.yaml` handles deployment to GitHub Pages.

#### `firebase/`
- Contains Firebase configuration files.
- Examples:
    - `firestore.rules`: Defines security rules for Firestore.
    - `firestore.indexes.json`: Defines Firestore indexes.
